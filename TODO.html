A4 - Jumbo multi-descriptor RX + Intel-style buffer/frame register split

  • [ ] split MAXFRM into RXBSZ (RX buffer size) + max frame length + CTRL.LPE (long packet enable), mirroring Intel RCTL.BSIZE / RLPML / LPE
  • [ ] RX supports a single packet spanning multiple descriptors (one buffer per segment, last sets EOP)
  • [ ] jumbo larger than one buffer is reassembled from multiple descriptors; oversized frames are dropped
  • Value: real-NIC scatter / multi-buffer RX, and the separation of "buffer size" from "max frame"
  • Depends on: single-buffer RX baseline done

A6 - Stateless offloads: checksum + TSO (+ LRO; GRO already in use)

  • [ ] checksum offload: advertise NETIF_F_HW_CSUM; RX mark CHECKSUM_UNNECESSARY (guest<->host is local/trusted), TX accept CHECKSUM_PARTIAL; the device "computes"/marks. Removes the per-byte software checksum the guest stack pays today (background-knowledge 7.7)
  • [ ] TSO + SG: advertise NETIF_F_SG|TSO; vnic_start_xmit passes the GSO super-skb (skb_is_gso) without skb_linearize, maps frags as multiple descriptors + a TSO command (mss, header len); the device TX engine segments into MSS frames (header template + per-segment seq/IP-id/checksum). Per-packet driver/descriptor/doorbell cost paid once per ~64 KB (bg 7.8)
  • [ ] LRO (optional): device coalesces same-flow RX segments before delivery (NETIF_F_LRO). GRO (software) is already active via napi_gro_receive - nothing to add for GRO
  • Value: the stateless-offload set real NICs expose via ethtool -K; checksum is foundational and unlocks TSO/LRO; TSO is per-packet amortization without jumbo (complements MTU)
  • Depends on: Level 2 baseline; checksum offload first (TSO requires it)

A5 - RX buffer Model B (page_pool / build_skb / copy-break)

  • [ ] post plain buffers / pages into the RX ring instead of pre-allocated skbs (D-18 rejects this for the baseline, which uses Model A)
  • [ ] build the skb at completion with build_skb / napi_build_skb; attach the page as a frag
  • [ ] page_pool recycling to avoid per-packet page alloc/free
  • [ ] copy-break: copy small frames into a small skb and recycle the buffer
  • [ ] batched / deferred RX refill (the enabler for the above): the clean loop only consumes and advances next_to_clean; refill runs separately in bursts at a threshold (e.g. ixgbe's IXGBE_RX_BUFFER_WRITE = 16) using a **separate next_to_use refill cursor**, ringingRDT**once per batch**. This is the RX analogue of TX doorbell batching (D-32). Today VN1000 refills each slot immediately in the same NAPI pass (onenext_to_clean`, simplest + correct).
  • Value / why ixgbe defers refill: (1) batch the RDT MMIO write - one PCIe write per ~16 descriptors instead of per packet (the main throughput win, same idea as TX D-32); (2) batch buffer alloc + DMA-map - amortise the per-call cost, better locality; (3) enable page reuse / page-split - decoupling "took the data" from "gave the device a new buffer" lets the refill phase decide per buffer whether to recycle the page (no new dma_map) or allocate; (4) keep the per-packet hot path tight. Cost: a second cursor + threshold logic + page-refcount bookkeeping, and refill must not lag enough to starve the device of buffers.
  • Depends on: Model A RX baseline (Phase 3) done
