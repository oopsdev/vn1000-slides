Documentation index

A guide to the docs/ set: what each file is, and the order to read them in. The project's goal is to understand the Linux NIC driver working flow end to end, so the docs split into a learning path (read top to bottom), build/run/debug SOPs, and reference assets.

New here? Open nic-driver-walkthrough.md first, or the slide deck.

1. Learning path (read in this order)

# Document What it is
1 nic-driver-walkthrough.md START HERE. The narrative tour: the mental model, the driver lifecycle (init/probe/open/close/remove), TX and RX end to end, the three levels + the one HAL seam, the hard parts (each mapped to a decision / lesson / background topic), and a repo map. Embeds the TX/RX path diagrams.
2 slides/vnic-slides.html Self-contained teaching slide deck (arrows / buttons / F fullscreen / page jump / PDF export). TCP/IP flow -> PCIe NIC anatomy + sample datasheet -> the Linux driver framework -> the QEMU device model -> the project goal -> dev/test setup -> the performance benchmark. View rendered: https://oopsdev.github.io/vn1000-slides/ (GitHub Pages mirror; this repo is private, so the source HTML here only renders when opened locally). The public mirror is auto-deployed by CI on every push that touches the deck or its diagrams - just edit & push; details in slides/README.md.
3 architecture.md System design: layering, the TX/RX flow diagrams, the interrupt/polling model, locking, and the Level 1 -> Level 2 evolution. The "what this driver does".
4 vn1000-datasheet.md The authoritative device hardware spec: PCI config space, the BAR0 register map, the descriptor format, the TX/RX ring buffers + the DD control flow, and the theory of operation. include/vnic_regs.h is derived from it.
5 background-knowledge.md The Linux kernel mechanisms behind the design (topics 1-11): the datapath, NAPI, deferred execution, memory barriers, MMIO, TUN/TAP, throughput, descriptor placement, the netdev lifecycle, interrupt registers, and DMA mappings. The "general kernel concepts".
6 design-decisions.md The decision log (D-01..D-33): for each choice, the decision / why / what was rejected.
7 lessons-learned.md Empirical findings from testing & tuning (L-01..L-12); raw data lives under results/.

2. Build / run / debug (SOPs)

Document What it is
dev-setup.md Environment setup SOP - WSL2 + an Ubuntu QEMU guest for kernel-module development.
qemu-build.md SOP to build a QEMU (pinned to the guest's version) with the VN1000 device, boot the guest under it, and set up the host tap / NAT.
qemu-device-model.md Level 2 background - how QEMU emulates the PCIe device (QOM / MMIO / DMA / MSI callbacks) and how the guest driver maps onto it.
debug-kernel.md SOP to run the test suite under KASAN / lockdep / kmemleak (the acceptance gate).
