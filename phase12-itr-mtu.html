Phase 12 - ITR moderation and MTU sweeps

Perf analysis of the two device levers beyond the TX doorbell (results/phase12-tx-doorbell.md): interrupt moderation (ITR, D-33) and MTU. Guest <-> host over the tap, QEMU 6.2 + KVM. See lessons-learned L-11 (ITR) and L-12 (MTU).

ITR sweep (scripts/itr-sweep.sh, iperf3 -R, MTU 1500)

itr_us   interrupts   ints/s     rx_drop   rx_throughput
0        8494         849        0         3.10 Gbits/sec
50       3894         389        0         2.79 Gbits/sec
500      1165         116        0         2.83 Gbits/sec
2000     149          14         0         2.65 Gbits/sec
5000     181          18         0         2.75 Gbits/sec

ITR cuts interrupts ~60x (849 -> 14/s) at flat throughput, 0 drop. NAPI is the primary moderator (itr=0 is already only ~849/s, not per-packet); ITR trims further by batching more per interrupt. No drops even at ~14 ints/s because NAPI keeps polling in softirq under load (the interrupt only restarts it after idle). See L-11. (Latency cost of ITR not yet measured - backlog.)

TX (iperf3, guest -> host, doorbell batching on, itr=0, DUR=30, two runs):

MTU    ints/s (r1/r2)   rx_drop   tx_throughput (r1 / r2)
1500   1304 / 768       0         3.22 / 3.33 Gbits/sec
3000   1920 / 1616      0         4.57 / 4.19 Gbits/sec
6000   2425 / 1493      0         6.56 / 6.22 Gbits/sec
9000   1024 / 1299      0         8.59 / 8.23 Gbits/sec

Throughput rises monotonically and reproducibly with MTU in both directions (~2.5x at 9000 vs 1500) - the per-packet cost is amortized over ~6x fewer packets. TX is faster than RX at jumbo (9000: ~8.4 vs ~7.4 Gbit/s): TX batches the doorbell (D-32) and the engine sweeps a whole burst per kick, while RX runs a per-frame receive() callback in the iothread. The interrupt rate does NOT track MTU (it swings 2-3x run-to-run at the same MTU): it is governed by NAPI completion timing, not packet count. 0 drop. A single 10 s run was too noisy (a spurious 9000 throughput dip + wild interrupt swings); 30 s x 2 was needed to separate signal from noise. See L-12.

Note: an earlier 10 s RX run showed 1500 3.46 / 3000 4.47 / 6000 5.01 / 9000 3.71 Gbit/s - the 9000 dip did not reproduce at 30 s (7.4 Gbit/s both runs), confirming it was variance.
