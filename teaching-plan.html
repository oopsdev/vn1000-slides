Weekly teaching plan

A 13-week, milestone-per-week schedule for teaching this project to students over the summer, from 2026-06-23 to 2026-09-15 (mid-September). Students are on summer vacation and work on the project full-time, so each weekly milestone is sized to one full-time week of effort, and the schedule is deliberately tight.

  • Weekly meeting: every Tuesday 08:00-09:00 (Taiwan time, UTC+8) - 1 hour, synchronous.
  • Cadence: the meeting is a checkpoint + lecture + goal-set, not the lab itself. Students build full-time between meetings; each meeting (1) reviews last week's deliverable, (2) teaches the week's concept from the slides, (3) hands out the next lab.
  • Shape: the core driver is finished by W8 - Level 1 by W4, Level 2 (real tap NIC) by W7, performance + demo by W8. The summer's remaining time (W9-W13) is spent on the advanced backlog (A1-A6) as real curriculum, finishing with an integration + report week.
  • Finishing a milestone early just means starting the next one early - the plan is front-to-back work, not a fixed weekly drip.

Section numbers are slides in slides/vnic-slides.html (rendered: https://oopsdev.github.io/vn1000-slides/); "bg N" = a topic in background-knowledge.md; "D-xx" / "L-xx" = entries in design-decisions.md / lessons-learned.md; "Phase N" / "A-N" = ../TODO.md.

Week-by-week detail

W1 - 06-23 - Recap, anatomy, environment, and the skeleton

  • Meeting: course intro; the TCP/IP packet journey + the driver's stack<->hardware role (1.1-1.4); PCIe NIC anatomy + the VN1000 datasheet (2.1-2.4); the project goal (5.1).
  • Concepts: slides 1.1-1.4, 2.1-2.4, 5.1; vn1000-datasheet.md; bg 8.
  • Lab (-> W2): Phase 0 (WSL2 + QEMU guest, hello module - dev-setup.md) and Phase 1 (repo layout, vnic_regs.h, fake registers, alloc_etherdev + register_netdev, net_device_ops).
  • Checkpoint: guest boots over SSH; hello module loads; vnic0 appears in ip link.

W2 - 06-30 - The TX and RX datapath

  • Meeting: the driver framework + HAL seam (D-12); TX end to end (skb -> descriptor -> TDT doorbell -> completion; DD ownership + NTU/NTC); RX with pre-posted skbs (Model A, D-18), the internal IP-swap loopback, napi_gro_receive, refill.
  • Concepts: slides 3.1, 3.2.1/3.2.2, 3.3.1/3.3.2, 5.2; bg 1.
  • Lab (-> W3): Phase 2 (TX path) + Phase 3 (RX path + NAPI + loopback).
  • Checkpoint: ping <peer> works through the loopback; TX/RX stats rise.

W3 - 07-07 - Interrupts, NAPI completion, and hardening

  • Meeting: the interrupt registers + top/bottom-half split; NAPI schedule/complete/re-arm; memory barriers + the DD handshake; ring concurrency (spin_lock_bh, the stop/wake invariant).
  • Concepts: slides 3.5, 3.7; bg 2 (NAPI), bg 4 (barriers), bg 10 (interrupt registers); D-20/21/23.
  • Lab (-> W4): Phase 4 (tasklet-simulated MSI, ICR/ICS/IMS/IMC) + Phase 5.1/5.2 (lock discipline, DMA map-error D-22, the leak/UAF/teardown walk).
  • Checkpoint: iperf lossless (0 Retr) over the MSI path (L-03); lock/lifecycle audit done.

W4 - 07-14 - Level 1 complete: MTU, tests, and the acceptance gate

  • Meeting: DMA mappings (coherent vs streaming, dma_mapping_error, bounce buffer, unmap-before-read - bg 11); ndo_change_mtu; teardown ordering (bg 12); the KASAN + lockdep + kmemleak gate.
  • Concepts: slides 3.6, 3.8; bg 11, bg 12; debug-kernel.md.
  • Lab (-> W5): Phase 5.3 (MTU) + run tests/ (01-06) + the KASAN/lockdep/kmemleak gate; read the L2 intro ahead.
  • Milestone: Level 1 is functionally complete and verified.

W5 - 07-21 - Level 2: the QEMU device model, MMIO and MSI

  • Meeting: how QEMU emulates the PCIe device - QOM TypeInfo, PCIDevice, MemoryRegionOps (MMIO), msi_init/msi_notify; building it into a pinned QEMU (D-27).
  • Concepts: slides 4, 5.3, 5.4; qemu-device-model.md, qemu-build.md; bg 5.
  • Lab (-> W6): Phase 8 - the device skeleton + the L2 probe (pci_register_driver, pci_iomap, dma_set_mask, pci_alloc_irq_vectors, request_irq).
  • Checkpoint: lspci shows 1234:11e9; MMIO read-back; MSI self-test passes.

W6 - 07-28 - Level 2: the DMA datapath

  • Meeting: the real DMA datapath - dma_alloc_coherent (ring) + dma_map_single (buffers); the device walking TDH..TDT with pci_dma_read/write; portable DMA (unmap before deliver, D-26).
  • Concepts: slide 5.4; bg 10, bg 11 applied; D-25/26.
  • Lab (-> W7): Phase 9 - run tests/run-all.sh on L2 over real MMIO/DMA/MSI.
  • Checkpoint: the Level 1 suite passes on L2 via device loopback (L-07).

W7 - 08-04 - Level 2 complete: the real tap backend

  • Meeting: the tap backend - qemu_new_nic, qemu_send_packet, can_receive/receive; host<->guest connectivity; TUN/TAP background.
  • Concepts: slide 5.5; bg 6 (TUN/TAP); D-30.
  • Lab (-> W8): Phase 10/11 - tap setup + the VNIC_TAP= run variant; verify ping + tcpdump.
  • Milestone: Level 2 is complete - vnic0 is a real pingable NIC (L-08).

W8 - 08-11 - Performance and the demo

  • Meeting: real DMA vs L1 memcpy; TX doorbell moderation (D-32, ~4.5x); device ITR (D-33, ~60x fewer interrupts); the MTU sweep; per-packet vs per-byte cost + VM-exits.
  • Concepts: slides 7, 8; bg 7; L-10/11/12.
  • Lab (-> W9): Phase 12 - scripts/{itr,perf}-sweep.sh; record the demo (ping + tcpdump + benchmark).
  • Checkpoint: benchmark table produced; demo recorded. Core project complete.

W9 - 08-18 - A6: stateless offloads

  • Meeting: checksum offload (NETIF_F_HW_CSUM), TSO/GSO (segment) and LRO/GRO (coalesce); why checksum is foundational and unlocks TSO.
  • Concepts: slide 3.9; bg 7.7-7.9; TODO A6.
  • Lab (-> W10): advertise + implement checksum offload, then TSO + SG (device-side segmentation); confirm GRO is already active.
  • Checkpoint: ethtool -k vnic0 shows the offloads; the TSO path moves a GSO super-skb correctly.

W10 - 08-25 - A3: MSI-X, multi-queue, and RSS

  • Meeting: single MSI -> MSI-X multi-vector; multiple TX/RX queues; RSS (hashing flows to queues/CPUs).
  • Concepts: bg 2, bg 10; TODO A3.
  • Lab (-> W11): MSI-X vectors + N TX/RX rings + per-queue NAPI; an RSS hash to steer flows.
  • Checkpoint: multi-queue traffic spreads across CPUs (/proc/interrupts, ethtool -S).

W11 - 09-01 - A2 + A1: mode comparison and Interrupt Auto-Mask

  • Meeting: forced INT / POLL / ADAPTIVE modes (the interrupt-vs-polling trade-off); IAM - read ICR auto-masks the cause (atomic ack + mask), removing the ICR->IMC race.
  • Concepts: bg 2, bg 3, bg 10; TODO A2, A1.
  • Lab (-> W12): a MODE register + the three-mode benchmark; an IAM register + the simplified ISR.
  • Checkpoint: a three-mode throughput/CPU/latency table; the IAM ack+mask works.

W12 - 09-08 - A4 + A5: jumbo multi-descriptor RX and page_pool

  • Meeting: splitting MAXFRM into buffer-size vs max-frame + LPE; one frame spanning multiple RX descriptors; RX buffer Model B (page_pool / build_skb / copy-break) + batched refill.
  • Concepts: bg 1, bg 8; TODO A4, A5.
  • Lab (-> W13): multi-descriptor (scatter) RX with reassembly; build_skb on pooled pages + a batched RDT refill cursor; copy-break for small frames.
  • Checkpoint: scatter RX reassembles jumbo frames; refill is batched; copy-break recycles buffers.

W13 - 09-15 - Integration, retrospective, and report

  • Meeting: put it together; review the design-decision and lessons logs end to end; what a production NIC adds beyond this.
  • Lab: harden, document, and write up the project; final demo.
  • Checkpoint: final demo + a written report/presentation. Buffer week for any slipped work.

If the pace slips

The hard deliverables are L1 (W4) and L2 + tap (W7); performance + demo (W8) close out the core. If the build weeks (W1-W8) slip, absorb it in the advanced weeks (W9-W13) - those are independent backlog items and any one can be dropped or shortened. W13 is an explicit buffer. If the class is ahead, deepen an advanced item (e.g. real RSS hashing, or a full page_pool recycle path) rather than adding scope to the core.
