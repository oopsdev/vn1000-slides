Slides — deployment

The teaching deck lives here as vnic-slides.html. It renders only when opened locally in this repo, because the repo is private. The rendered version the world sees is a separate public repo, oopsdev/vn1000-slides, served by GitHub Pages at https://oopsdev.github.io/vn1000-slides/.

Deployment is automatic

You do not sync anything by hand. Just edit the source and push to main:

  • vnic-slides.html — the deck, and/or
  • diagrams/*.svg — any embedded diagram

.github/workflows/deploy-slides.yml then rebuilds the public mirror and pushes it (only when something actually changed). GitHub Pages rebuilds ~1–2 min later; hard-refresh (Ctrl+Shift+R) to bust the browser's SVG cache.

How it works (maintainers)

The mirror is a flattened copy of the deck — everything sits at the repo root so Pages can serve it. The workflow builds it on every push that touches docs/slides/*.html or docs/diagrams/**:

source (this repo) mirror (vn1000-slides) transform
docs/slides/vnic-slides.html index.html rewrite diagrams/ → diagrams/
docs/diagrams/*.svg diagrams/*.svg verbatim copy
docs/slides/*.html (the rest) <name>.html (flat) rewrite diagrams/ → diagrams/ and the deck link → index.html

The non-deck docs/slides/*.html are the rendered documentation set - generated from the Markdown docs by scripts/gen-docs-site.py (which reuses scripts/gen-docs-site.py). It renders every interlinked doc (docs/*.md, README/TODO/CHANGELOG, results/*.md, tests/README.md) to a flat <name>.html with cross-doc .md links rewritten to the sibling .html (docs/index.md → docs-index.html; the three README.md → <dir>-readme.html). Regenerate after editing any doc (python scripts/gen-docs-site.py), then commit the .html. Links to source files (driver/*.c, scripts/*.sh) are left as-is and are dead on the public mirror (this repo is private).

Cross-repo push uses a deploy key: the public half is a write-enabled deploy key on vn1000-slides; the private half is the MIRROR_DEPLOY_KEY Actions secret in this repo. To rotate it, regenerate an ed25519 keypair, replace the deploy key on vn1000-slides, and re-set the secret. You can also trigger the workflow manually from the Actions tab (Run workflow → workflow_dispatch).
